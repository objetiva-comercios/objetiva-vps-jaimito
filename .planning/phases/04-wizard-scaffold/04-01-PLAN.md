    **Paso 3: Crear `cmd/jaimito/setup/wizard.go`**
    Package `setup`. Implementar:

    `SetupData` struct con campos:
    - `ConfigPath string`
    - `ExistingCfg *config.Config` (nil si no existe)
    - `ConfigErr error` (error de carga si config es invalido)
    - `Mode string` ("new", "edit", "fresh") — determinado por detect-config step (plan 02)

    `Step` interface:
    ```go
    type Step interface {
        Init(data *SetupData) tea.Cmd
        Update(msg tea.Msg, data *SetupData) (Step, tea.Cmd)
        View(data *SetupData) string
        Done() bool
    }
    ```

    `WizardModel` struct:
    - `steps []Step`
    - `currentStep int`
    - `data *SetupData`
    - `quitting bool`
    - `confirmExit bool`

    `stepNames` — slice de 7 strings con los nombres visibles de los steps:
    ```go
    var stepNames = []string{
        "Bienvenida",
        "Bot Token",
        "Canal General",
        "Canales Extra",
        "Servidor",
        "Base de Datos",
        "Resumen",
    }
    ```

    `WizardModel.Init()` retorna `(tea.Model, tea.Cmd)` — llama `m.steps[0].Init(m.data)`.

    `WizardModel.Update()`:
    - USAR `tea.KeyPressMsg` (NO `tea.KeyMsg` — v2 API, ver Pitfall W3)
    - `ctrl+c`: si `confirmExit` es true, `m.quitting=true` + `tea.Quit`; sino `confirmExit=true`
    - Cualquier otra tecla cuando `confirmExit==true`: reset `confirmExit=false`
    - `esc` o `up`: si `currentStep > 0`, retroceder (valores se mantienen)
    - Delegar al step activo. Si `step.Done()` y hay mas steps, avanzar.

    `WizardModel.View()` retorna `tea.View`:
    - Si `quitting`: "Saliendo...\n"
    - Si `confirmExit`: mostrar mensaje "Seguro que queres salir? Se pierden los datos ingresados. (s/n)" con ErrorStyle
    - Sino: `renderLayout(m)` — sidebar izquierda + separador "│" + contenido del step derecha + barra de atajos inferior

    `renderSidebar(currentStep int, completedSteps map[int]bool)`:
    - Itera `stepNames`, renderiza con StepDone "✓ " si completado, StepActive "▸ " si actual, StepPending "  " si pendiente
    - Agrega contador `[N/7]` con HintStyle debajo

    `renderLayout(m WizardModel)`:
    - Sidebar a la izquierda (ancho fijo ~20 chars)
    - Separador vertical "│" con ColorGray
    - Contenido del step a la derecha
    - Barra inferior con hints segun contexto: "Enter: continuar │ Esc: volver │ Ctrl+C: salir"

    `FormatNonInteractiveError() string`:
    - Retorna string formateado con ErrorStyle (rojo)
    - Texto: "Error: jaimito setup requiere una terminal interactiva.\n\nSi estas usando curl | bash, el instalador ya redirige stdin automaticamente.\nPara ejecutar manualmente: jaimito setup --config /etc/jaimito/config.yaml"

    `RunWizard(cfgPath string, existingCfg *config.Config, configErr error) error`:
    - Crea SetupData con los parametros
    - Crea WizardModel con los steps (en esta fase: solo WelcomeStep; los demas son stubs placeholder)
    - Maneja stdout no-TTY con Pitfall W2: abrir /dev/tty si stdout no es terminal, setear lipgloss renderer
    - `tea.NewProgram(model, opts...)` + `p.Run()`

    `NewWizardModel(cfgPath string, existingCfg *config.Config, configErr error) WizardModel`:
    - Construye SetupData
    - Inicializa steps slice: `[]Step{&WelcomeStep{}}` (plan 02 agrega DetectConfigStep)
    - Steps 2-7 son placeholder stubs que solo muestran "Proximamente..." y Done()=true con Enter
