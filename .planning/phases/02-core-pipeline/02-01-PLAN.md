<task type="auto">
  <name>Task 2: Add schema migration 002 and message DB operations</name>
  <files>internal/db/schema/002_nullable_title.sql, internal/db/messages.go</files>
  <action>
Create internal/db/schema/002_nullable_title.sql:
- SQLite cannot ALTER COLUMN, so use shadow-table approach:
  1. `ALTER TABLE messages RENAME TO messages_old;`
  2. `CREATE TABLE messages (...)` — same schema as 001 but `title TEXT` (nullable, no NOT NULL).
  3. `INSERT INTO messages SELECT * FROM messages_old;`
  4. `DROP TABLE messages_old;`
  5. Recreate indexes: `idx_messages_status` and `idx_messages_created_at`.

Create internal/db/messages.go in package `db`:

1. Define `Message` struct: ID string, Channel string, Priority string, Title *string (pointer for nullable), Body string, Tags []string (JSON-decoded), Metadata map[string]any (JSON-decoded), Status string, CreatedAt string, UpdatedAt string.

2. `EnqueueMessage(ctx context.Context, db *sql.DB, id string, channel string, priority string, title *string, body string, tags []string, metadata map[string]any) error`:
   - JSON-marshal tags ([]string -> string) if non-nil, else sql.NullString.
   - JSON-marshal metadata (map -> string) if non-nil, else sql.NullString.
   - INSERT INTO messages (id, channel, priority, title, body, tags, metadata, status) VALUES (?, ?, ?, ?, ?, ?, ?, 'queued').
   - title parameter is *string — pass as-is (nil becomes SQL NULL).

3. `GetNextQueued(ctx context.Context, db *sql.DB) (*Message, error)`:
   - SELECT * FROM messages WHERE status = 'queued' ORDER BY created_at ASC LIMIT 1.
   - If no rows, return nil, nil.
   - Unmarshal tags JSON string back to []string. Unmarshal metadata JSON to map[string]any.
   - Use sql.NullString for nullable columns (title, tags, metadata).

4. `MarkDispatching(ctx context.Context, db *sql.DB, id string) error`:
   - UPDATE messages SET status = 'dispatching', updated_at = datetime('now') WHERE id = ?.

5. `MarkDelivered(ctx context.Context, db *sql.DB, id string) error`:
   - UPDATE messages SET status = 'delivered', updated_at = datetime('now') WHERE id = ?.

6. `MarkFailed(ctx context.Context, db *sql.DB, id string) error`:
   - UPDATE messages SET status = 'failed', updated_at = datetime('now') WHERE id = ?.

7. `RecordAttempt(ctx context.Context, db *sql.DB, messageID string, attempt int, sendErr error) error`:
   - Determine status: if sendErr == nil then "success", else "error".
   - Determine errorText: if sendErr != nil then sendErr.Error(), else empty string (sql.NullString).
   - INSERT INTO dispatch_log (message_id, attempt, status, error) VALUES (?, ?, ?, ?).

All functions take *sql.DB as parameter (not methods on a custom type) — consistent with Phase 1 pattern (ReclaimStuck takes *sql.DB).
  </action>

<task type="auto">
  <name>Task 3: Add API key DB operations with shared hash helper</name>
  <files>internal/db/apikeys.go</files>
  <action>
Create internal/db/apikeys.go in package `db`:

1. `HashToken(token string) string`:
   - Exported helper used by both SeedKeys and the auth middleware.
   - `hash := sha256.Sum256([]byte(token))` then `hex.EncodeToString(hash[:])`.
   - Import crypto/sha256 and encoding/hex.

2. `LookupKeyHash(ctx context.Context, db *sql.DB, keyHash string) (string, error)`:
   - SELECT id FROM api_keys WHERE key_hash = ? AND revoked = 0.
   - If no rows (sql.ErrNoRows), return "", nil (not found is not an error).
   - Returns key ID (UUID) if found.

3. `SeedKeys(ctx context.Context, db *sql.DB, keys []config.SeedAPIKey) error`:
   - Import github.com/chiguire/jaimito/internal/config and github.com/google/uuid.
   - For each key in keys:
     a. Validate key.Key starts with "sk-" (return error if not).
     b. Compute hash via HashToken(key.Key).
     c. Generate UUID v4 for the key ID: uuid.New().String().
     d. INSERT OR IGNORE INTO api_keys (id, key_hash, name) VALUES (?, ?, ?).
   - INSERT OR IGNORE makes this idempotent — safe to run every startup.
   - Log at Info level each newly seeded key (by name, never log the raw key value).

4. `UpdateLastUsed(ctx context.Context, db *sql.DB, keyID string) error`:
   - UPDATE api_keys SET last_used_at = datetime('now') WHERE id = ?.
   - This is called fire-and-forget from the auth middleware (non-critical).
  </action>
