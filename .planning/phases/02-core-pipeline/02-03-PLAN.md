<task type="auto">
  <name>Task 2: Create dispatcher with polling loop, retry, and 429 handling</name>
  <files>internal/dispatcher/dispatcher.go</files>
  <action>
Create internal/dispatcher/dispatcher.go in package `dispatcher`:

Import: context, database/sql, fmt, log/slog, time, github.com/go-telegram/bot, github.com/go-telegram/bot/models, github.com/chiguire/jaimito/internal/config, github.com/chiguire/jaimito/internal/db, github.com/chiguire/jaimito/internal/telegram.

1. `Start(ctx context.Context, database *sql.DB, b *bot.Bot, cfg *config.Config)`:
   - Create `time.NewTicker(1 * time.Second)` — polls every 1 second per CONTEXT.md.
   - Spawn goroutine with ticker loop:
     - On tick: call `dispatchNext(ctx, database, b, cfg)`. Log errors at Error level.
     - On ctx.Done(): return (stop dispatcher).
   - Defer `ticker.Stop()` inside goroutine.

2. `dispatchNext(ctx context.Context, database *sql.DB, b *bot.Bot, cfg *config.Config) error`:
   - Call `db.GetNextQueued(ctx, database)`.
   - If nil message returned, return nil (nothing to dispatch).
   - Call `dispatch(ctx, database, b, cfg, msg)`.

3. `dispatch(ctx context.Context, database *sql.DB, b *bot.Bot, cfg *config.Config, msg *db.Message) error`:
   - Constants: `maxAttempts = 5`, `baseDelay = 2 * time.Second`.

   - Loop from attempt 1 to maxAttempts:
     a. Call `db.MarkDispatching(ctx, database, msg.ID)`.
     b. Format message: `text := telegram.FormatMessage(msg)`.
     c. Resolve chat ID: `chatID := cfg.ChatIDForChannel(msg.Channel)`.
        - If chatID == 0, mark failed and return error (channel not found — should not happen if notify handler validated).
     d. Send: `_, err := b.SendMessage(ctx, &bot.SendMessageParams{ChatID: chatID, Text: text, ParseMode: models.ParseModeMarkdown})`.
        - CRITICAL: `models.ParseModeMarkdown` = "MarkdownV2" string. Do NOT use `ParseModeMarkdownV1` (= "Markdown", the old V1 format).
     e. Record attempt: `db.RecordAttempt(ctx, database, msg.ID, attempt, err)`.
     f. If err == nil:
        - `db.MarkDelivered(ctx, database, msg.ID)`
        - `slog.Info("message delivered", "id", msg.ID, "channel", msg.Channel, "attempt", attempt)`
        - Return nil.
     g. If `bot.IsTooManyRequestsError(err)`:
        - Type-assert: `tooMany := err.(*bot.TooManyRequestsError)`
        - `waitDur := time.Duration(tooMany.RetryAfter) * time.Second`
        - `slog.Warn("rate limited by Telegram", "retry_after_sec", tooMany.RetryAfter, "attempt", attempt, "id", msg.ID)`
        - Wait with cancellation: `select { case <-time.After(waitDur): case <-ctx.Done(): return ctx.Err() }`
        - Continue to next attempt.
     h. For other errors:
        - If attempt < maxAttempts:
          - `delay := baseDelay * time.Duration(1<<uint(attempt-1))` — 2s, 4s, 8s, 16s.
          - `slog.Warn("telegram send failed, retrying", "attempt", attempt, "delay", delay, "error", err, "id", msg.ID)`
          - Wait with cancellation: `select { case <-time.After(delay): case <-ctx.Done(): return ctx.Err() }`

   - After loop exhausted (all 5 attempts failed):
     - `db.MarkFailed(ctx, database, msg.ID)`
     - `slog.Error("message delivery failed after max attempts", "id", msg.ID, "attempts", maxAttempts)`
     - Return nil (per CONTEXT.md: "mark message as failed silently — logged, no further action").

Per CONTEXT.md locked decisions:
- 5 retry attempts with exponential backoff
- Respect Telegram 429 retry_after exactly
- After 5 exhausted retries, mark failed silently
- Dispatcher polls every 1 second
- Sequential dispatch — one message at a time, preserves order
