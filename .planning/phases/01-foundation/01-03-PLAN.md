  <name>Task 1: Create Telegram validation package and systemd unit file</name>
  <files>internal/telegram/client.go, systemd/jaimito.service</files>
  <action>
    **1. Create `internal/telegram/client.go`:**

    - Import `github.com/go-telegram/bot`

    - **`ValidateToken(ctx context.Context, token string) (*bot.Bot, error)`**:
      - Call `bot.New(token)` — this automatically calls getMe with a default 5s timeout
      - If error, return: `fmt.Errorf("invalid telegram bot token: %w", err)`
      - Return the *bot.Bot instance (reused for chat validation and later in Phase 2)

    - **`ValidateChats(ctx context.Context, b *bot.Bot, channels []config.ChannelConfig) error`**:
      - Import config types from `internal/config`
      - Deduplicate chat_ids: use `map[int64]bool` since multiple channels can share the same chat_id (per CONTEXT.md)
      - For each unique chat_id, call `b.GetChat(ctx, &bot.GetChatParams{ChatID: ch.ChatID})`
      - If any getChat fails, return error with channel name: `fmt.Errorf("channel %q: chat_id %d unreachable: %w", ch.Name, ch.ChatID, err)`
      - IMPORTANT: ValidateToken MUST be called before ValidateChats (getChat will fail with auth error if token is wrong)

    Note on ChatID type: Check the actual type in `bot.GetChatParams` at implementation time. It may need type conversion. Telegram group/supergroup IDs are negative int64. The config stores `int64`.

    **2. Create `systemd/jaimito.service`:**

    Per CONTEXT.md locked decisions:
    ```ini
    [Unit]
    Description=jaimito VPS notification hub
    After=network.target
    ConditionPathExists=/usr/local/bin/jaimito

    [Service]
    Type=simple
    ExecStart=/usr/local/bin/jaimito
    Restart=on-failure
    RestartSec=5s
    StandardOutput=journal
    StandardError=journal
    SyslogIdentifier=jaimito

    [Install]
    WantedBy=multi-user.target
    ```

    - Type=simple: binary runs in foreground
    - Restart=on-failure: clean exits (os.Exit(0)) stay down, crashes restart
    - Runs as root (no User= line) per CONTEXT.md
    - Logs to stdout captured by journald (StandardOutput=journal)
    - Binary at /usr/local/bin/jaimito per CONTEXT.md
  </action>

<task type="auto">
  <name>Task 2: Wire main.go with full startup sequence and graceful shutdown</name>
  <files>cmd/jaimito/main.go</files>
  <action>
    Replace the stub main.go from Plan 01 with the full startup sequence.

    **Startup sequence (STRICT ORDER per RESEARCH.md):**

    1. **Initialize logging:** `slog.SetDefault(slog.New(slog.NewTextHandler(os.Stdout, &slog.HandlerOptions{Level: slog.LevelInfo})))` — logs to stdout for journald capture.

    2. **Parse config path flag:** Accept `-config` flag with default `/etc/jaimito/config.yaml`. Example: `flag.StringVar(&configPath, "config", "/etc/jaimito/config.yaml", "path to config file")`

    3. **Load config:** `cfg, err := config.Load(configPath)` — fail fast, exit 1 with slog.Error on failure.

    4. **Set up signal context:** `ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGTERM, syscall.SIGINT)` and `defer stop()`

    5. **Validate Telegram token:** `tgBot, err := telegram.ValidateToken(ctx, cfg.Telegram.Token)` — fail fast, exit 1. Log success: "telegram bot validated".

    6. **Validate Telegram chats:** `err = telegram.ValidateChats(ctx, tgBot, cfg.Channels)` — fail fast, exit 1. Log success: "telegram chats validated" with count.

    7. **Open database:** `database, err := db.Open(cfg.Database.Path)` — fail fast, exit 1. `defer database.Close()`.

    8. **Apply schema:** `err = db.ApplySchema(database)` — fail fast, exit 1. Log: "database schema applied".

    9. **Reclaim stuck messages:** `reclaimed, err := db.ReclaimStuck(ctx, database)` — fail fast, exit 1. If reclaimed > 0, log at INFO: "reclaimed stuck messages" with count. If 0, no log needed.

    10. **Start cleanup scheduler:** `cleanup.Start(ctx, database, 24*time.Hour)` — runs in background goroutine.

    11. **Log ready:** `slog.Info("jaimito started", "channels", len(cfg.Channels), "db", cfg.Database.Path)`

    12. **Wait for shutdown:** `<-ctx.Done()` then `slog.Info("shutting down")` — deferred database.Close() runs.

    **Error handling pattern:** For each step that can fail:
    ```go
    if err != nil {
        slog.Error("step description", "error", err)
        os.Exit(1)
    }
    ```
    Use `os.Exit(1)` only BEFORE defers are set up. After `defer database.Close()` is registered, use a different pattern or ensure the error exits happen before that defer. Actually, since we want fail-fast and the deferred close is just cleanup, `os.Exit(1)` after the defer is acceptable here — the OS will close the file handle anyway. But for cleanliness, structure the code so critical validations (config, telegram) happen before `db.Open`, and DB-related failures after the defer are handled gracefully.

    **Import paths** (will match whatever module path Plan 01 chose):
    - `internal/config`
    - `internal/db`
    - `internal/cleanup`
    - `internal/telegram`
  </action>
