<task type="auto">
  <name>Task 2: Create config package with Load and Validate</name>
  <files>internal/config/config.go, configs/config.example.yaml</files>
  <action>
    Create `internal/config/config.go` with:

    1. **Types** (per CONTEXT.md locked decisions):
       - `Config` struct with fields: `Telegram TelegramConfig`, `Database DatabaseConfig`, `Channels []ChannelConfig`
       - `TelegramConfig` struct: `Token string` (yaml:"token")
       - `DatabaseConfig` struct: `Path string` (yaml:"path") — default `/var/lib/jaimito/jaimito.db`
       - `ChannelConfig` struct: `Name string` (yaml:"name"), `ChatID int64` (yaml:"chat_id"), `Priority string` (yaml:"priority")

    2. **Load(path string) (*Config, error)** function:
       - `os.ReadFile(path)` — if file missing, return error with clear message: "config file not found: %s"
       - `yaml.Unmarshal(data, &cfg)` — if malformed, return parse error
       - Set defaults: if `cfg.Database.Path` is empty, set to `/var/lib/jaimito/jaimito.db`
       - Call `cfg.Validate()` before returning

    3. **Validate() error** method on Config:
       - Telegram token must not be empty → error: "telegram.token is required"
       - At least one channel required → error: "at least one channel is required"
       - A channel named "general" must exist → error: "a channel named 'general' is required"
       - Each channel: name must not be empty, chat_id must not be zero, priority must be one of "low", "normal", "high" → error: "channel %q: invalid priority %q (must be low, normal, or high)"
       - Duplicate channel names rejected → error: "duplicate channel name: %q"

    4. **Example config** at `configs/config.example.yaml`:
       - `telegram.token`: placeholder "YOUR_BOT_TOKEN_HERE"
       - `db.path`: "/var/lib/jaimito/jaimito.db"
       - All 7 channels pre-defined (general, deploys, errors, cron, system, security, monitoring) with placeholder chat_id values and sensible default priorities (general=normal, errors=high, deploys=normal, cron=low, system=normal, security=high, monitoring=normal)

    Per CONTEXT.md: No env var overrides. All credentials inline. Strict validation — unknown priorities rejected. `general` channel required.
  </action>
  <verify>

Verify validation catches: missing token, missing general channel, invalid priority. Run with `go test ./internal/config/...` or a quick main test.

    Alternatively: `go vet ./internal/config/...` passes and `go build ./internal/config/...` succeeds.
  </verify>
