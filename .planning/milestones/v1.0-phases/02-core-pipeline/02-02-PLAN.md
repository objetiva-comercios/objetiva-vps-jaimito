**Step 2: Create internal/api/response.go** — shared response helpers:

1. `WriteJSON(w http.ResponseWriter, status int, data any)`:
   - Set `Content-Type: application/json`.
   - Call `w.WriteHeader(status)` BEFORE `json.NewEncoder(w).Encode(data)`.
   - CRITICAL: WriteHeader MUST come before Encode, otherwise Go auto-sends 200.

2. `WriteError(w http.ResponseWriter, status int, message string)`:
   - Calls WriteJSON with `ErrorResponse{Error: message}`.

3. Define `ErrorResponse` struct with `Error string` json tag.

**Step 3: Create internal/api/middleware.go** — bearer auth middleware:

1. `BearerAuth(database *sql.DB) func(http.Handler) http.Handler`:
   - Returns a chi-compatible middleware function.
   - Extract `Authorization` header.
   - Check `strings.HasPrefix(authHeader, "Bearer ")` — if missing, WriteError 401 "missing or invalid Authorization header".
   - Extract token via `strings.TrimPrefix(authHeader, "Bearer ")`.
   - Check `strings.HasPrefix(token, "sk-")` — if not, WriteError 401 "invalid token format".
   - Hash token: `keyHash := db.HashToken(token)` (import internal/db package).
   - Lookup: `keyID, err := db.LookupKeyHash(r.Context(), database, keyHash)`.
   - If err != nil or keyID == "", WriteError 401 "invalid token".
   - Fire-and-forget: `go db.UpdateLastUsed(context.Background(), database, keyID)` — use background context since the request context may be cancelled.
   - Call `next.ServeHTTP(w, r)`.

**Step 4: Create internal/api/server.go** — chi router:

1. `NewRouter(database *sql.DB, cfg *config.Config) http.Handler`:
   - `r := chi.NewRouter()`
   - Global middleware: `middleware.RequestID`, `middleware.RealIP`, `middleware.Logger`, `middleware.Recoverer`, `middleware.Timeout(30 * time.Second)`.
   - Health endpoint (unauthenticated): `r.Get("/api/v1/health", HealthHandler)`.
   - Authenticated group via `r.Group()`:
     - `r.Use(BearerAuth(database))`
     - `r.Post("/api/v1/notify", NotifyHandler(database, cfg))`
   - Return r.

Note: NewRouter does NOT take *bot.Bot — the API layer only enqueues messages into the DB. The dispatcher reads from the DB and sends to Telegram. This clean separation means the API has no direct Telegram dependency.

Create internal/api/handlers.go:

**1. Request/Response types:**

```go
type NotifyRequest struct {
    Title    *string        `json:"title"`
    Body     string         `json:"body"`
    Channel  string         `json:"channel"`
    Priority string         `json:"priority"`
    Tags     []string       `json:"tags,omitempty"`
    Metadata map[string]any `json:"metadata,omitempty"`
}

type NotifyResponse struct {
    ID string `json:"id"`
}

type HealthResponse struct {
    Status  string `json:"status"`
    Service string `json:"service"`
}
```

**2. NotifyHandler(database *sql.DB, cfg *config.Config) http.HandlerFunc:**

- Limit body: `r.Body = http.MaxBytesReader(w, r.Body, 1<<20)` (1MB).
- Decode JSON: `json.NewDecoder(r.Body).Decode(&req)`. On error, WriteError 400 "invalid JSON body".
- Apply defaults per CONTEXT.md locked decisions:
  - `if req.Channel == ""` then `req.Channel = "general"`
  - `if req.Priority == ""` then `req.Priority = "normal"`
- Validate:
  - `req.Body == ""` → WriteError 400 "body is required"
  - `!cfg.ChannelExists(req.Channel)` → WriteError 400 `fmt.Sprintf("unknown channel: %q", req.Channel)`
  - `!validPriority(req.Priority)` → WriteError 400 `fmt.Sprintf("invalid priority: %q", req.Priority)`
- Define `validPriority(p string) bool` as unexported function — checks against "low", "normal", "high".
- Generate UUID v7: `id, err := uuid.NewV7()`. On error, WriteError 500 "failed to generate message ID".
- Enqueue: `db.EnqueueMessage(r.Context(), database, id.String(), req.Channel, req.Priority, req.Title, req.Body, req.Tags, req.Metadata)`. On error, WriteError 500 "failed to enqueue message".
- Respond: `WriteJSON(w, http.StatusAccepted, NotifyResponse{ID: id.String()})`.
- Import github.com/google/uuid.

**3. HealthHandler(w http.ResponseWriter, r *http.Request):**

- `WriteJSON(w, http.StatusOK, HealthResponse{Status: "ok", Service: "jaimito"})`.
