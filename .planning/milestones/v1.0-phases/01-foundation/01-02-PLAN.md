  <action>
    **1. Create `internal/db/schema/001_initial.sql`:**

    ```sql
    CREATE TABLE IF NOT EXISTS messages (
        id          TEXT PRIMARY KEY,
        channel     TEXT NOT NULL,
        priority    TEXT NOT NULL DEFAULT 'normal',
        title       TEXT NOT NULL,
        body        TEXT NOT NULL,
        tags        TEXT,
        metadata    TEXT,
        status      TEXT NOT NULL DEFAULT 'queued',
        created_at  TEXT NOT NULL DEFAULT (datetime('now')),
        updated_at  TEXT NOT NULL DEFAULT (datetime('now'))
    );

    CREATE INDEX IF NOT EXISTS idx_messages_status ON messages(status);
    CREATE INDEX IF NOT EXISTS idx_messages_created_at ON messages(created_at);

    CREATE TABLE IF NOT EXISTS dispatch_log (
        id          INTEGER PRIMARY KEY AUTOINCREMENT,
        message_id  TEXT NOT NULL REFERENCES messages(id),
        attempt     INTEGER NOT NULL DEFAULT 1,
        status      TEXT NOT NULL,
        error       TEXT,
        attempted_at TEXT NOT NULL DEFAULT (datetime('now'))
    );

    CREATE INDEX IF NOT EXISTS idx_dispatch_log_message_id ON dispatch_log(message_id);

    CREATE TABLE IF NOT EXISTS api_keys (
        id          TEXT PRIMARY KEY,
        key_hash    TEXT NOT NULL UNIQUE,
        name        TEXT NOT NULL,
        created_at  TEXT NOT NULL DEFAULT (datetime('now')),
        last_used_at TEXT,
        revoked     INTEGER NOT NULL DEFAULT 0
    );
    ```

    Status values for messages: queued, dispatching, delivered, failed.
    Status values for dispatch_log: success, failed, retrying.
    api_keys table created now (empty) — Phase 3 populates it.

    **2. Create `internal/db/db.go`:**

    - Import `modernc.org/sqlite` driver with blank import: `_ "modernc.org/sqlite"`
    - Import `embed`, `database/sql`, `github.com/adlio/schema`

    - `//go:embed schema/*.sql` to embed the migration files into `var schemaFS embed.FS`

    - **`Open(path string) (*sql.DB, error)`** function:
      - Build DSN: `"file:" + path + "?_pragma=journal_mode(WAL)&_pragma=busy_timeout(5000)&_pragma=foreign_keys(1)&_pragma=synchronous(NORMAL)"`
      - CRITICAL: Use `_pragma=journal_mode(WAL)` format (parentheses, NOT equals). This is `modernc.org/sqlite` format, NOT `mattn/go-sqlite3`.
      - Call `sql.Open("sqlite", dsn)` — driver name is "sqlite" for modernc
      - Set `db.SetMaxOpenConns(1)` — single-writer pool, prevents SQLITE_BUSY
      - Set `db.SetMaxIdleConns(1)`
      - Set `db.SetConnMaxLifetime(0)` — connections live for app lifetime
      - Ping the database to verify connection: `db.Ping()`
      - Ensure parent directory exists: `os.MkdirAll(filepath.Dir(path), 0755)` before opening
      - Return `(*sql.DB, error)`

    - **`ApplySchema(db *sql.DB) error`** function:
      - Use `schema.FSMigrations(schemaFS, "schema/*.sql")` to load migrations
      - Use `schema.NewMigrator(schema.WithDialect(schema.SQLite))` for SQLite dialect
      - Call `migrator.Apply(db, migrations)`
      - Idempotent — safe on every startup (adlio/schema tracks applied migrations)

    - **`ReclaimStuck(ctx context.Context, db *sql.DB) (int64, error)`** function:
      - Execute: `UPDATE messages SET status = 'queued', updated_at = datetime('now') WHERE status = 'dispatching'`
      - Return `result.RowsAffected()` count
      - This handles crash recovery — dispatcher may have crashed mid-flight (PERS-03)
      - Caller logs the count at INFO level

    Note: `Open` creates the parent directory automatically per CONTEXT.md "auto-creates SQLite DB file and schema if not present (zero manual setup)".

  <action>
    Create `internal/cleanup/scheduler.go` with:

    1. **`Start(ctx context.Context, db *sql.DB, interval time.Duration)`** function:
       - Run an initial purge immediately (so cleanup happens even if service rarely restarts — per RESEARCH.md recommendation)
       - Start a goroutine with `time.NewTicker(interval)` (caller passes 24*time.Hour or configurable)
       - On each tick, call `purgeOldMessages(ctx, db)`
       - On `ctx.Done()`, stop ticker and return — clean shutdown
       - Log errors from purge at ERROR level using `slog.Error`
       - Log successful purge count at INFO level: "purged old messages" with "delivered" and "failed" counts

    2. **`purgeOldMessages(ctx context.Context, db *sql.DB) error`** (unexported):
       - Execute two DELETE statements (or one combined):
         ```sql
         DELETE FROM dispatch_log WHERE message_id IN (
           SELECT id FROM messages
           WHERE (status = 'delivered' AND created_at < datetime('now', '-30 days'))
              OR (status = 'failed' AND created_at < datetime('now', '-90 days'))
         );
         DELETE FROM messages
         WHERE (status = 'delivered' AND created_at < datetime('now', '-30 days'))
            OR (status = 'failed' AND created_at < datetime('now', '-90 days'));
         ```
       - IMPORTANT: Delete dispatch_log entries FIRST (foreign key constraint), then messages
       - Wrap in a transaction for atomicity
       - Return error if either DELETE fails

    PERS-04 specifies: delivered > 30 days, failed > 90 days.
